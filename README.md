# Web Manual Testing — Urban Routes

## Project Overview
This project contains a complete manual QA testing process performed on the Urban Routes web application (TripleTen training environment).

The goal was to validate core functionality through structured testing, regression validation, and bug reporting.

## Tools Used
- Chrome DevTools
- Jira (Bug Tracking)
- Documentation in Markdown / Google Sheets
- Git & GitHub

## Testing Types Performed
- Functional Testing
- Regression Testing
- Exploratory Testing
- UI Validation
- Bug Reporting & Documentation

## Project Structure
- `docs/` → test plan, requirements, and summary report
- `test-cases/` → test cases and regression checklist
- `bug-reports/` → bug reports with reproduction steps
- `evidence/` → screenshots of findings

## Note
This project was tested in the TripleTen training environment. The server may reset periodically, but all test artifacts and documentation remain valid.
