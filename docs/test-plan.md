## 1. Objective
The objective of this testing effort is to validate the functionality, stability, and usability of the Urban Routes web application through structured manual testing.

This test plan focuses on ensuring that core user flows work as expected and that critical defects are identified and documented.

---

## 2. Scope

### In Scope
- UI validation and layout consistency
- Core functional flows
- Input validation
- Error handling behavior
- Regression testing of main features

### Out of Scope
- Performance testing
- Security testing
- Load testing
- Backend/database validation

---

## 3. Test Strategy
Testing was performed using:
- Functional manual testing
- Exploratory testing
- Regression checklist execution
- Bug reporting with clear reproduction steps
- Evidence collection (screenshots)

---

## 4. Test Environment
- Application: Urban Routes (TripleTen training environment)
- Browser: Google Chrome
- Tools: Chrome DevTools, Jira, GitHub
- OS: Windows / macOS

---

## 5. Test Deliverables
- Test cases documentation
- Regression checklist
- Bug reports
- Test summary report

---

## 6. Entry Criteria
- Application is accessible
- Requirements and expected behavior are provided
- Test environment is stable enough to execute tests

---

## 7. Exit Criteria
- All planned test cases executed
- Regression checklist completed
- Critical and high severity defects documented
- Final summary report delivered

---

## 8. Risks
- Server resets may affect reproducibility of issues
- Some test data may not persist between sessions
- Limited access to backend logs

---

## 9. Approval
This test plan is created as part of QA training and documentation practice and is intended to demonstrate manual testing methodology.
